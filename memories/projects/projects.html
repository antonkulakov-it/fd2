Минимальные требования

Должны быть установлены: node.js

Должны быть установлены: bower, gulp

Инициализируем проект

Создаем папку, в которой будет создан проект (#$ mkdir -p /path/to/dir/)

Заходим в нее (#$ cd /path/to/dir)

Инициализируем серверную часть (#$ npm init), отвечаем на вопросы.

Устанавливаем bower

Проверяем, нужна ли установка bower (#$ bower). Если не выдало сообщений об ошибках, установка не нужна

Если нужно, устанавливаем bower. Глобально: (#$ npm install -g bower). Локально: (#$ npm install bower).

Инициализируем frontend часть (#$ bower init), отвечаем на вопросы.

Если нужно, устанавливаем пакеты (#$ bower install [pack_name])

Устанавливаем gulp

Проверяем, нужна ли установка gulp (#$ gulp). Если выдало сообщение что не найден файл gulpfile.js, или нет ошибок - установка не нужна.

Создаем gulpfile.js (#$ touch gulpfile.js)

Создаем таски в gulpfile.js

Если необходимо установить какие-то плагины для gulp - устанавливаем npm install [plugin_name]

Инициализируем git

Инициализируем git (локальный репозиторй) в нашей папке (#$ git init)

Создаем файл .gitignore, туда заносим, как правило: node_modules, bower_components, test, tests, dist

Добавляем удаленный репозиторий (#$ git remote add [local_name] [addres_of_remote_repo])

Посмотреть список удаленных репозиториев (#$ git remote -v])

Делаем первый коммит (#$ git add [some_file] && git commit)

Делаем отправку в удаленный репозиторий (#$ git push [addres_of_remote_repo] [branch])

Забираем из удаленного репозитория (#$ git pull [addres_of_remote_repo] [branch])

Работка с ветками разработки

Посмотреть список веток (#$ git branch)

Создать новую ветку (#$ git branch [branch_name])

Переключиться на ветку (#$ git checkout [branch_name])

Делаем отправку ветки в удаленный репозиторий (#$ git push [addres_of_remote_repo] [branch_name])

Забираем ветку из удаленного репозитория (#$ git pull [addres_of_remote_repo] [branch_name])

Слияние веток можно производить только после того, как сделали коммит.

Слияние ветки [some_dev] с веткой [some_branch]: (#$ git checkout [some_branch] && git merge [some_dev])

В разрешении конфликтов слияний вам поможет google/yandex :)
